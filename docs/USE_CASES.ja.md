# ユースケース

このページは、プロジェクトの意味は分かったけれど「実際に何に使えばいいのか」を知りたい人向けです。

## 1. OSSのIssue整理

複数のClaude Code CLI画面で、同じIssueを別々の視点から調べます。再現担当、コード確認担当、テスト案担当、メンテナー返信担当に分けられます。

良い出力: 再現できたか、怪しいコード箇所、次にやること、返信案。

## 2. PRレビュー

動作、テスト、安全性、互換性、ドキュメント、リリースノートを別々の画面で確認します。1つの長いチャットより、レビュー観点を分けやすくなります。

良い出力: マージ可否、ブロッカー、リスク、追加確認。

## 3. リリース準備

変更履歴、互換性、スモークテスト、ドキュメント、リリースノート、フォローアップを分担します。

良い出力: タグを切る前に確認できるリリースチェックリスト。

## 4. 機能アイデア出し

ユーザーの流れ、実装案、CLI設計、テスト方針を別々に考えさせ、最後に比較します。

良い出力: 採用案と却下案が分かる意思決定メモ。

## 5. 調査とアイデア比較

複数の視点から調査結果を集め、1つの方向に決める前に比較します。

良い出力: 比較表と、次に進めるべき案。

## まず試すなら

迷ったら Issue整理 から始めるのがおすすめです。結果が分かりやすく、複数画面で作業する意味も体感しやすいです。
