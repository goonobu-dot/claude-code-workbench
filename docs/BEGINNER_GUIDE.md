# Beginner-Friendly Guide

This page explains Claude Code Workbench in plain language.

## What this project is

Claude Code Workbench is a local tool for opening several Claude Code CLI sessions in one terminal window.

It lets you split a larger task into smaller roles:

- one pane researches the topic
- one pane compares ideas
- one pane plans implementation
- one pane checks risks
- one pane writes the final summary

All panes point at the same local idea folder, so the work stays organized.

## Why it exists

One AI chat can help, but complex work often needs multiple viewpoints.

This project gives Claude Code CLI a repeatable multi-pane workspace for research, planning, comparison, and OSS maintenance work.

## What it can help with

- Researching a topic from several angles
- Collecting feature ideas
- Comparing implementation plans
- Reviewing risks before acting
- Saving reusable Markdown notes
- Running 4, 6, or 9 Claude Code panes against one workspace

## Simple analogy

Imagine a group study session.

One person checks facts, one person writes notes, one person asks critical questions, and one person summarizes the answer.

Claude Code Workbench gives Claude Code CLI that kind of team-like setup. The human user still decides what to trust and what to do next.

## What it does not do

- It does not run directly inside GitHub.
- It does not publish your local Mac files.
- It does not automatically merge code.
- It is not affiliated with Anthropic.

It is a local workspace for organizing AI-assisted research and planning.
